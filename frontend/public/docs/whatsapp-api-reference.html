Textzy WhatsApp API Reference

Professional public and tenant-authenticated WhatsApp messaging reference covering plain public sends, templates, media, flows, webhooks, and operational API examples.

1. Public WhatsApp API GET and POST

Use the public WhatsApp API when an external system needs to send a plain WhatsApp text message through Textzy using tenant-specific API credentials.

GET Example

GET https://textzy-backend-production.up.railway.app/api/public/messages/send
  ?recipient=919999999999
  &msg=Hello from Textzy WhatsApp API
  &user=MONEYART
  &pswd=YOUR_PASSWORD
  &apikey=YOUR_API_KEY
  &channel=whatsapp
  &tenantSlug=moneyart

POST Example

{
  "recipient": "919999999999",
  "message": "Hello from Textzy WhatsApp API",
  "user": "MONEYART",
  "password": "YOUR_PASSWORD",
  "apiKey": "YOUR_API_KEY",
  "tenantSlug": "moneyart",
  "channel": "whatsapp",
  "idempotencyKey": "wa-20260308-0001"
}
Public WhatsApp API examples are intended for plain text sends. Richer capabilities such as templates, interactive messages, media, and flow execution use authenticated tenant APIs.

2. Required Fields

  • recipient: mobile number with country code
  • message or msg: plain text message
  • tenantSlug: tenant slug
  • user: tenant API username
  • password or pswd: tenant API password
  • apiKey or apikey: tenant API key
  • channel: whatsapp

3. Responses

Success

{
  "jobId": "5d64f8bf-4c1f-4e59-92a9-4a0f5a8c992e",
  "message": "Accepted"
}

Error

{
  "message": "Invalid authorization.",
  "code": "401"
}

4. Authenticated Tenant WhatsApp API

Use authenticated tenant APIs when messages originate from Textzy web, mobile, desktop, inbox, or workflow execution.

Session Message

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "body": "Hello from Textzy",
  "useTemplate": false
}

Template Message

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "useTemplate": true,
  "templateName": "order_update",
  "templateLanguageCode": "en",
  "templateParameters": ["John", "#1234"]
}

Interactive Buttons

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "isInteractive": true,
  "interactiveType": "button",
  "body": "How can we help you?",
  "interactiveButtons": ["Support", "Sales", "Accounts"]
}

Interactive Flow

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "isInteractive": true,
  "interactiveType": "flow",
  "body": "Open the flow below.",
  "interactiveFlowId": "FLOW_ID",
  "interactiveFlowCta": "Open",
  "interactiveFlowAction": "navigate",
  "interactiveFlowScreen": "start",
  "interactiveFlowDataJson": "{}",
  "interactiveFlowMessageVersion": 3
}
Session messaging is subject to WhatsApp conversation-window rules. Use approved templates when a session message is not allowed.

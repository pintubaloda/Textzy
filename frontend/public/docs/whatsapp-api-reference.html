Textzy WhatsApp API Reference

Professional tenant-facing reference for Textzy WhatsApp Business messaging, approved templates, media upload, webhook processing, automation flows, Meta flow management, customer request examples, and tenant analytics. Platform-only operator APIs are intentionally excluded.

1. Authentication and Tenant Context Tenant session

WhatsApp operational APIs are tenant-scoped and intended for authenticated Textzy sessions using httpOnly cookie transport. Login can include authenticator verification when 2FA is enabled.

  • POST /api/auth/login
  • POST /api/auth/two-factor/verify-login
  • GET /api/auth/projects
  • POST /api/auth/switch-project
  • GET /api/auth/me
  • POST /api/auth/refresh
Use authenticated tenant APIs for WhatsApp send, templates, inbox, analytics, and automation. Tenant context comes from the selected project, not from caller-supplied public credentials. Public simple URL mode is intended for SMS only, and platform-only operator endpoints are not part of this reference.

2. Complete Working Flow

  1. Login to Textzy
  2. If enabled, complete authenticator verification
  3. Load projects and switch to the target tenant
  4. Check WABA readiness
  5. Send session, template, interactive, or flow message
  6. Monitor inbox, webhook status, and analytics

4. Session and Template Messaging

Session Message

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "body": "Hello from Textzy",
  "useTemplate": false
}

Template Message

{
  "recipient": "919999999999",
  "channel": "WhatsApp",
  "useTemplate": true,
  "templateName": "order_update",
  "templateLanguageCode": "en",
  "templateParameters": ["John", "#1234"]
}
Session sends work only inside the active 24-hour WhatsApp customer window. Outside that window, use an approved template. These writes depend on authenticated tenant session and CSRF protection in first-party clients.

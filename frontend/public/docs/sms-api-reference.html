Textzy SMS API Reference

Professional public SMS integration reference for tenant credentials, DLT-compliant sends, sender and template registries, delivery reporting, and production request formats.

1. Public SMS API GET and POST

Use the public SMS API when an ERP, CRM, website, or backend service needs to send SMS directly through Textzy using tenant-specific API credentials.

GET Example

GET https://textzy-backend-production.up.railway.app/api/public/messages/send
  ?recipient=919999999999
  &msg=Your approved DLT message text
  &user=MONEYART
  &pswd=YOUR_PASSWORD
  &apikey=YOUR_API_KEY
  &channel=sms
  &sender=MNYART
  &PE_ID=1601100000000006533
  &Template_ID=1207171593687982329
  &tenantSlug=moneyart

POST Example

{
  "recipient": "919999999999",
  "message": "Your approved DLT message text",
  "user": "MONEYART",
  "password": "YOUR_PASSWORD",
  "apiKey": "YOUR_API_KEY",
  "tenantSlug": "moneyart",
  "channel": "sms",
  "sender": "MNYART",
  "peId": "1601100000000006533",
  "templateId": "1207171593687982329",
  "idempotencyKey": "sms-20260308-0001"
}
For India DLT traffic, the sender ID, PE_ID, template ID, and final SMS text must match approved records.

2. Required Fields

FieldUse
recipientMobile number with country code
message / msgFinal approved SMS text
tenantSlugTenant slug
userTenant API username
password / pswdTenant API password
apiKey / apikeyTenant API key
senderApproved sender ID
peId / PE_IDApproved entity ID
templateId / Template_IDApproved template ID
Each tenant has its own API username, password, and API key. Optional IP whitelist rules can also be enforced per tenant.

3. Responses

Success

{
  "jobId": "5d64f8bf-4c1f-4e59-92a9-4a0f5a8c992e",
  "message": "Accepted"
}

Error

{
  "message": "Invalid authorization.",
  "code": "401"
}

Common codes: 400, 401, 403, 422, 429, 503.

4. Registry APIs

Template Registry

  • GET /api/sms/templates
  • POST /api/sms/templates
  • PUT /api/sms/templates/{id}
  • DELETE /api/sms/templates/{id}
  • POST /api/sms/templates/{id}/status
  • POST /api/sms/templates/import-approved-csv

Sender Registry

  • GET /api/sms/senders
  • GET /api/sms/senders/stats
  • POST /api/sms/senders
  • PUT /api/sms/senders/{id}
  • DELETE /api/sms/senders/{id}

5. DLT Compliance

  • Use only approved sender IDs
  • Use only approved entity IDs
  • Use only approved template IDs
  • Final text must follow the approved template

6. Delivery Reporting

Tenant SMS reports and delivery ledgers can show statuses such as:

  • accepted
  • queued
  • sent
  • delivered
  • failed
  • rejected

7. Segment Logic

  • English GSM: 1-160 chars = 1 SMS, multipart uses 153 chars per segment
  • Unicode / Regional: 1-70 chars = 1 SMS, multipart uses 67 chars per segment
